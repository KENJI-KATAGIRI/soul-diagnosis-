# 意識レベル統合仕様

## 目的

このシステムは「診断」ではなく、「魂のナビ」によって**一致した意思決定**へ導くためのナビゲーションです。  
そのため、結果は評価ではなく、**次に進む方向提示**として扱います。

## 状態を決める3要素

1. **意識レベル（位置）**
2. **エネルギー（強さ）**
3. **一致度（ズレ）**

この3つの掛け合わせで、現在地を判断します。

## 意識レベルの扱い

- 数値の基準は一般的な意識レベル概念の既存スケールを内部参照する
- ただし、UIではその概念をそのまま見せず、**魂のナビの受信状態**として翻訳する

### UI上の翻訳

- **ノイズ状態**: 迷い・不安・停滞が前に出ている
- **受信しているが迷っている状態**: 感覚はあるが、まだ揺れている
- **一致状態**: 確信・直感・行動がつながりやすい

## 身体感覚との紐付け

- 低い状態: 体が縮む、重い、呼吸が浅い
- 中間状態: 揺れる、迷う、違和感が混ざる
- 高い状態: 緩む、広がる、呼吸が深い

## エネルギースコアとの統合

- `0〜30`: 不安・迷い領域
- `30〜60`: 過渡期
- `60〜80`: 一致直前
- `80〜100`: 一致状態

## 最重要ロジック: 一致度

重要なのは「レベルが高いか」ではなく、**その状態と行動が一致しているか**。

検出対象:

- 頭では理解しているが行動がズレている
- 感情と行動が一致していない
- エネルギーは高いが方向がズレている

## 実装場所

- `consciousness_mapper.py`
  - 意識レベル / エネルギー / 一致度の統合ロジック
- `nav_diagnosis_ai/result_logic.py`
  - 診断AIのシグナルから意識レベル推定
- `templates/nav_ai/result.html`
  - 6段構成の結果表示
- `templates/result.html`
  - 10問診断結果への簡易統合
- `soul_nav_ai/engine.py`
  - 対話プロンプトに「受信状態」と一致度の考え方を反映

## 結果表示の構造

1. 現在の状態
2. 意識レベルの位置
3. 身体感覚の説明
4. ズレの特定
5. 一致した場合の未来
6. 今やるべき一歩

## UX方針

- 抽象論ではなく体感ベースで説明する
- 難しい専門用語は避ける
- 数値だけで終わらせない
- 次の行動へ必ずつなげる
